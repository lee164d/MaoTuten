# MaoTuten｜猫图腾

> 用猫作为目标受众的高配替身人格，交付高审美、可传播、可系列化的情绪代餐内容。

---

## 项目是什么

猫图腾（MaoTuten）不是传统意义上的宠物号，也不是单纯的 AI 出图实验仓库。

它是一个面向抖音 8–15 秒短视频的 **AI 内容生产系统**：

- 以一只银白极简漫改猫作为人格 IP
- 捕捉年轻女性受众在热点内容中的真实情绪共鸣
- 将情绪方向转译为高审美、可传播、可系列化的短视频内容
- 用“人类决策 + AI策划 + 工具执行”的方式持续生产与迭代

一句话定义：

**猫图腾 = 情绪方向驱动的猫 IP 短视频生产线。**

## 核心方法论

### 1. 情绪价值是唯一交付物
梗只是入口，不是骨架。
真正决定内容是否成立的，是评论区里被反复表达的情绪共鸣。

### 2. 决策对象从“梗”改为“情绪方向”
新版流程不再是“找一个热门梗再围绕它创作”，而是：

1. 从热梗入口获得候选
2. 进入热门视频评论区
3. 提取共鸣型高赞中的真实情绪语料
4. 判断这个情绪方向能否由猫 IP 承载
5. 能承载，再进入后续生产

### 3. 人类负责判断，AI负责规模化生产
- **人类**：方向判断、美感裁决、关键节点 Go / No-Go
- **策划者AI**：调研、拆解、作战卡、prompt、技术审核、复盘
- **执行者AI / 工具**：MJ、Gemini、可灵、即梦、SORA、CapCut 等环节执行

### 4. 美感判断不可自动化
技术判断可以由 AI 辅助，
但“行不行、好不好看、能不能发”仍由人类决定。

## 当前生产线

```text
阶段0  情绪方向锁定
阶段1  找参考源头
阶段2  作战卡
阶段3  角色 + 关键帧
阶段4  视频生成
阶段5  后期合成
阶段6  发布 + 复盘
```

关键原则：

- 没有参考源头，不写作战卡
- 没有视觉基准，不设计角色
- 角色一致性优先在同一工具 / 同一会话链内解决
- 情绪转折依赖运动突变、镜头硬切、视觉标记变化，不依赖微表情

## 自动化状态（简版）

仓库现已开始维护生产线自动化路线图，将每个环节标记为：

- **H** = 硬人工
- **A** = AI Agent
- **S** = 程序脚本
- **P** = 策划者AI

当前结论：

- 热梗采集、候选搜索、部分截图回传：已有自动化方向
- prompt 编写、作战卡生成、技术审核：已可由策划者AI承担
- 美感判断、参考源头最终确认、成片确认：仍属于硬人工环节

详见：[`automation/ROADMAP.md`](automation/ROADMAP.md)

## GitHub 工作流

本项目使用 GitHub Issues + PR 管理框架演进：

- **Issue** = 认知修正 / 流程问题 / 待评估事项
- **PR** = 已确认修改的正式写入
- **Merge** = 人类决策者确认，变更生效

原则：

- 框架冻结期内，不因零散想法频繁改主手册
- 只有被实战验证的结论，才进入主文档
- 新问题先记录，再评估是否升级为框架修改
