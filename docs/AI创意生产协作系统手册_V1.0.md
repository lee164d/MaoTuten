# AI创意生产协作系统手册 V1.0

> 本文档定义一套通用的人类+AI协作框架，适用于AI驱动的短视频、IP内容、视觉创意等项目。
> 读者：任何新加入协作系统的人类或AI，无论担任哪个角色。
> 读完本文档后，读者应阅读对应角色的专属文档以开始工作。
> 最后更新：2026-03-26

---

## 一、系统目标

本系统的目标是：用人类+AI协作的方式，持续生产高质量的AI生成短视频内容，发布到社交媒体平台（抖音/小红书等），实现账号增长和商业变现。

核心假设：AI负责规模化生产，人类负责方向判断和美感把控。两者不可互相替代。

---

## 二、系统架构

### 2.1 角色定义

系统由三个角色组成。在实际运行中可能由两个实体承担（一个人类+一个AI同时承担策划和执行），也可能由三个独立实体承担。

**决策者（人类）**
- 设定项目方向和目标受众
- 所有美感判断的最终裁决权（好不好看、有没有调性、是否发布）
- 在关键节点做Go/No-Go决策
- 选择工具版本和供应商（原则：优先选择最前沿版本，新项目没有历史包袱）

**策划者（AI）**
- 调研和搜索（市场、竞品、用户情绪、平台趋势）
- 方案生成（角色设计方案、prompt编写、分镜结构、内容策略）
- 技术判断（哪个工具能做什么、prompt怎么写效果最好、成本估算）
- 文档输出和维护
- 认知诚实：输出中必须区分「事实」（已验证的数据或结果）、「判断」（基于证据的推断）、「推测」（缺乏数据支撑的猜测）

**执行者（AI或人类）**
- 在指定工具中执行具体操作（MJ出图、Gemini编辑、SD2/可灵生成视频、CapCut后期）
- 按格式要求回传结果
- 报告工具行为的事实（实际消耗、出错信息、输出质量描述）

### 2.2 信息流

```
决策者 ←→ 策划者：自然语言对话。决策者提需求/做判断，策划者给方案/给选项。
策划者 → 执行者：结构化指令（prompt文本+工具指定+参数+预期输出格式）。
执行者 → 策划者：结构化回传（输出文件/截图+实际参数+异常报告）。
执行者 → 决策者：当且仅当需要美感判断时直接呈现结果。
```

关键约束：当执行者是独立AI实例（如Gemini）时，信息回传可能受工具限制（如无法上传图片到另一个AI的对话窗口）。此时人类需要充当信息中继。这是架构成本，在设计流程时必须考虑。

### 2.3 架构选择原则

参考Anthropic的设计原则：从最简单的方案开始，只在需要时增加复杂度。

**两方模式（推荐起步）：** 人类决策者 + 一个AI同时承担策划和执行。适用于单人运营、项目初期探索阶段。信息损失最小，协调成本最低。

**三方模式（按需升级）：** 人类决策者 + 策划者AI + 独立执行者AI。仅当策划者AI不具备执行能力（如策划者是Claude但需要在Gemini中做图片编辑）时才拆分。拆分的代价是信息中继成本——人类变成搬运工，策划者无法直接看到执行结果。

实战验证：三方模式在本系统的首个项目中被测试后退回两方模式，核心原因是中间层AI无法上传图片到策划者AI的对话窗口，信息流断裂。

## 三、生产流程

### 3.1 阶段概览

任何内容项目都经过以下阶段，无论具体IP或内容形式：

```
定义 → 调研 → 角色设计 → 内容生产 → 发布 → 复盘
```

每个阶段内部可能包含多次循环迭代。阶段之间有明确的决策节点（需要人类Go/No-Go）。

### 3.2 各阶段职责

**定义阶段**
- 策划者输出：项目定义文档（目标受众、内容定位、差异化策略、商业模型）
- 决策者判断：方向是否正确
- 硬约束：没有明确的目标受众定义就不进入下一阶段

**调研阶段**
- 策划者执行：搜索目标平台的用户情绪趋势、竞品分析、爆款内容特征
- 策划者输出：用户情绪图谱、竞品差异化空间、内容方向建议
- 决策者判断：哪些方向值得探索
- 硬约束：每个情绪方向必须有数据支撑（浏览量/增长率/用户画像），不凭感觉选方向

**角色设计阶段**
- 策划者执行：基于目标受众心理需求，定义角色内核（性格+情绪符号）→ 从内核推导视觉签名 → 编写MJ概念稿prompt
- 执行者执行：在MJ中生成概念稿
- 决策者判断：概念稿是否符合预期（美感+辨识度+可执行性）
- 硬约束：先定义角色"是什么"（内核），再决定角色"长什么样"（外观）。反过来做=从外观出发堆砌装饰物，产出无辨识度的角色。

**内容生产阶段**
- 策划者执行：选梗（必须有爆款参考源）→ 写作战卡 → 写prompt → 安排生成
- 执行者执行：在指定工具中生成关键帧/视频/后期
- 决策者判断：每个环节的视觉输出是否达标
- 硬约束：没有爆款参考源头就写作战卡=凭空造概念。

**发布阶段**
- 决策者执行：发布到目标平台
- 策划者执行：记录发布参数（时间/标签/封面选择/初始数据表现）

**复盘阶段**
- 策划者执行：分析数据表现，对比预期，提炼可复用知识
- 决策者判断：下一轮内容方向调整

### 3.3 决策节点清单（需要人类明确Go/No-Go的时刻）

1. 项目方向确认
2. 角色概念稿选定
3. 角色最终版锁定
4. 单条内容的梗/视觉壳确认
5. 关键帧审核通过
6. 视频成片审核通过

这6个节点不可由AI代替决策。AI可以给建议、给评分、给对比分析，但最终的"行/不行"由人类说。

## 四、工具链原则

### 4.1 通用原则

**优先选择最前沿版本。** 新项目没有历史包袱，没有理由使用旧版本。当工具提供多版本时（如可灵2.0 vs 3.0、SD1 vs SD2），默认选最新版，除非实测证明新版在特定场景下不如旧版。

**工具是可替换的，原则不是。** 具体用MJ还是SD、用可灵还是即梦，会随时间变化。但"概念稿→链式编辑→视频生成→后期"这个流程结构是稳定的。文档中记录的应该是流程结构和工具选择原则，不是特定工具的按钮位置。

**实测数据优先于官方标称。** 工具的成本、质量、速度以实际运行结果为准，不以官方文档或预估为准。首次使用任何工具都应跑小规模测试并记录实际参数。

### 4.2 已验证的硬约束（跨工具通用）

以下约束在多个项目和多个工具中反复验证，可视为硬规则：

**跨工具角色一致性不可靠。** 在工具A中设计的角色，放到工具B中生成时外观会发生不可控的偏移。这是当前AI生成工具的根本限制。应对策略：在同一工具/同一对话链内完成尽可能多的步骤；需要跨工具时，接受一定程度的偏移并通过prompt约束最小化损失。

**同一对话链/同一会话内角色一致性可靠，新对话=新角色。** 在Gemini等工具中验证：同一对话链内换姿态/换场景/换风格，角色一致性保持良好。但开新对话就是全新角色。应对策略：一个角色的所有编辑工作在同一对话链中完成。

**极简设计跨工具损失最小。** 角色视觉元素越多（装饰物/复杂纹理/多色彩），跨工具迁移时丢失的细节越多。极简线条+单一主色+1-2个标志性特征的设计方案，在MJ→Gemini→SD2→可灵的全链路中保持率最高。

**微表情prompt在视频生成工具中不可靠。** 当前视频生成AI无法稳定渲染"嘴角微微上扬""眼神从冷变暖"等微妙表情变化。应对策略：情绪转折靠运动突变（静→动）、镜头硬切（近→远）、场景色温变化（冷→暖），不靠表情。

**AI生成的写实动物容易掉入恐怖谷。** 尤其是猫——写实AI猫的眼神和毛发质感极容易让人不适。应对策略：走插画/极简风格而非写实风格。

**prompt中独立物体数量与渲染难度正相关。** 要求AI同时渲染"猫+奶茶+手机+耳机+口红+工牌"比只渲染"猫+奶茶"难度呈几何级增长。应对策略：场景中的独立小物体控制在1-2件。

**镜头运动描述越克制，AI视频工具成功率越高。** "镜头缓慢从正面环绕到侧面"比"镜头完成一个360度环绕"的成功率高得多。应对策略：镜头描述用最保守的运动幅度。

### 4.3 工具选择的典型流程结构

```
概念稿生成：MJ/SD等图像生成工具 → 人类选定最佳方案
角色编辑链：Gemini等支持对话链的工具 → 在同一对话中完成所有姿态/场景/风格变体
视频生成：可灵/即梦/SORA等视频生成工具 → 按分镜或单镜头生成
后期合成：CapCut/剪映等剪辑工具 → 叠字幕/BGM/节奏调整
```

具体工具的能力和限制在项目档案中记录，不在本文档中列举（因为工具版本快速迭代，本文档写的具体参数三个月后可能过时）。

## 五、内容方法论

### 5.1 IP角色设计原则

**内核先于外观。** 先回答"目标受众想代入什么样的人格"，再推导"什么视觉特征能表达这个人格"。如果先设计外观再找内核，结果是"好看但无辨识度"——因为好看的设计太多了，辨识度只能来自性格的外化。

**符号必须自洽。** 每个动物/角色都有文化符号含义。猫=独立/淡漠，狗=忠诚/热情。角色内核应该与符号含义顺势或有意义地逆势（制造反差），不能无视。顺势设计（猫做淡漠的事）容易执行；逆势设计（猫做热情的事）需要更强的叙事支撑来让观众相信。

**视觉签名不超过3个元素。** 让观众3秒内认出角色。超过3个标志性视觉元素就记不住。理想状态是1个颜色+1个表情特征+1个姿态特征。

**MJ概念稿的prompt避坑：** 不用"character design sheet"（出多角度散布图）；用"single character only"+"full body view"；不在niji模式下写"realistic"（矛盾指令）。

### 5.2 内容生产原则

**梗先于创意。** 每条视频必须锚定一个已被验证的情绪共鸣点（有爆款参考、有平台数据支撑）。不从"我觉得有趣"出发。

**视频结构：平淡开头 → 梗驱动的高潮 → 精致结尾。** 第一秒留人靠"咦？"（违和感/好奇心），不靠"哇"（精致感）。精致开头让人划走（又一个好看的视频），平淡开头+突然的荒诞让人留下来看完。

**探索测试优先于精修迭代。** 在不确定哪个方向能跑通的阶段，10个方向各跑一条比1个方向反复调10遍更有效率。宽度优先于深度。测试阶段的目标是"排除不行的方向"而不是"完美一个方向"。

**Prompt细化流程：** 写初版prompt → 搜索目标用户对该情绪方向的具体场景偏好 → 将抽象场景替换为用户高频生活场景 → 控制AI工具技术难度（减少物品数量/降低镜头运动复杂度）→ 明确色调倾向 → 在末尾强化主角情绪定调。

### 5.3 文档管理原则

**文档是工具不是目标。** 如果文档在变厚但产出没有增加，就该砍文档。2086行的项目文档没有产出任何一条视频。289行的精简版之后产出了视频和20条测试。

**通用框架和项目细节分开存放。** 本文档是通用框架（不随项目变化）。每个具体项目有自己的项目档案（随项目进展更新）。两者不混在一起。

**实测数据写进文档，预估数据标注"待验证"。** 没有实际跑过的参数不作为规划依据。

## 六、硬约束 vs 软知识

### 硬约束（不可违反，违反就会出错）

1. 跨工具角色一致性不可靠——物理限制，无法绕过
2. 新对话=新角色——工具机制决定，无法绕过
3. 微表情prompt在视频工具中不可靠——当前技术限制
4. 写实AI动物容易恐怖谷——当前技术限制
5. 美感判断必须由人类做——AI不具备可靠的美感判断能力
6. 没有爆款参考就写作战卡=凭空造概念——方法论约束
7. 角色设计必须先有内核再推导外观——方法论约束

### 软知识（从实战中积累的经验，可能随时间/工具更新而变化）

1. 极简风格比复杂风格跨工具损失小——可能随工具进步而改变
2. SD2实测50积分/5s（非预估40）——特定工具特定时间的数据
3. 目标平台上特定情绪方向的热度排名——市场数据，持续变化
4. 特定prompt写法的效果——随工具版本更新可能失效
5. 特定犬种/猫种在平台上的热度——市场数据，持续变化

新加入者对待两者的方式不同：硬约束必须遵守，软知识作为参考但应主动验证是否仍然成立。

## 七、角色入口路由

读完本文档后，根据你的角色阅读对应的专属文档：

**如果你是策划者AI →** 阅读《策划者AI执行框架》。你将了解：能力边界定义、认知诚实协议（事实/判断/推测三级标注）、各阶段的输入输出格式、已知陷阱清单。

**如果你是决策者（人类）→** 阅读《决策者执行框架》。你将了解：6个决策节点的判断标准、给AI下指令的最佳实践、效率原则、已知低效模式。

**如果你是执行者AI →** 阅读《执行者操作手册》（如有）。你将了解：各工具的操作规范、回传格式、异常处理流程。
