# 猫图腾·视觉叙事规范 V1.0

## 文档性质
本文档用于沉淀猫图腾在静态画面 / 关键帧中的视觉叙事规则，是《猫图腾_生产线手册_V1.0.md》的执行级补充文档。

它不替代生产线手册，不负责定义阶段流程；它只回答一个问题：

**一张画面该如何通过物件、角色和环境，把情绪状态精准地交付出来。**

当前版本的已验证适用范围：
- 赛道1｜独立自足型
- 赛道1子变体｜边界感疗愈
- 其他“状态型内容”可参考，但尚未视为跨赛道通用公式

---

## 核心原理

**画面中的每个物件都是角色心理状态的外化。**

如果一个物件不服务于情绪传递，它就不应该存在于画面中。因为它不会增加情绪值，只会分散观众对情绪的感知，变成噪音。

换句话说：
- 物件不是“让画面更丰富”的装饰
- 物件是“让情绪更精确”的函数变量
- 画面不是生活切片的堆叠，而是心理状态的空间化呈现

这条规则与主手册的以下原则一致：
- 情绪价值是唯一交付物
- 短视频密度指标：情绪密度 > 信息密度 > 情节密度
- 猫适配的判断标准是：能否把一个情绪状态视觉化成静态画面

## 当前已验证公式：三层情绪结构

### 前景｜制造张力
放置一个“**本应被关注但被无视**”的物件。

作用：
- 传递“有事情要处理，但我选择不管”
- 让画面出现外界索取与角色态度之间的轻张力
- 为“我不回应”提供可视化对象

常见类型：
- 亮着的手机 / 未读消息
- 被推开的外卖咖啡杯
- 未收拾的小事物
- 本应催促人的提醒性物件

原则：
- 只保留 0-1 个主要张力物件
- 它必须可见，但不能抢走角色主导权
- 它的价值在于“被无视”，不是“本身好看”

### 中景｜传递态度
猫的侧影 / 背影 / 松散体态是画面的情绪核心。

作用：
- 传递“我什么都不做，而且我不觉得这有什么问题”
- 承接前景张力，并用身体状态表达主动的不回应
- 让“毫不费力的不在乎”从抽象人格落到可见姿态

常见表达：
- 背对前景物件
- 重心歪向一边
- 一只前爪自然搭着
- 尾巴自然垂落 / 不居中
- 侧面 / 背影，不给正脸

原则：
- 姿态必须松，不端正，不对称，不摆拍
- 角色不是“没看到”，而是“看到了但不理”
- 对猫图腾来说，中景承担人格核，永远高于物件核

### 后景｜替换节奏
用与前景物件形成反差的安静环境，替换掉“被催促”的节奏。

作用：
- 传递“世界在运转，我不参与”
- 让画面从“外界索取”切换到“属于自己的时间”
- 给观众提供可以投射进去的情绪容器

常见类型：
- 黄昏天空 / 窗边自然光
- 缓慢变化的天空颜色
- 无人打扰的空环境
- 自然元素或极简留白

原则：
- 后景必须与前景张力形成节奏反差
- 后景的任务不是讲信息，而是提供“慢”
- 它应当支撑代入感，而不是增加额外叙事负担

## 猫图腾与万有真理的差异化

### 万有真理的已验证公式
- 前景：空 / 悦己小物件 → “没有人需要你”
- 中景：人物背影做无关紧要的事 → “无事可做且不焦虑”
- 后景：自然元素 → “时间在缓慢流动”

### 猫图腾的差异化升级
猫图腾不只做“本来就没有人找我”，而更适合做：

**“有事情存在，但我选择不care。”**

因此猫图腾在赛道1中的前景，常可从“空”升级为：
- 有但被无视的物件
- 会索取注意力但未被回应的信号

这让猫图腾比纯独居氛围图多了一层态度：
- 不是自然无事
- 而是主动不回应

这更贴近猫图腾主角的内核：
**毫不费力的不在乎。**

## 执行原则

### 1. 先写“观众应该感到什么”，再写画面元素
正确顺序：
1. 观众第一秒应该感到什么状态
2. 前景 / 中景 / 后景分别如何服务这个状态
3. 再把它翻译成物理画面

错误顺序：
- 先罗列场景里有什么
- 再希望这些物件“自动形成情绪”

### 2. 每个元素都必须回答：它在改变什么情绪输出？
如果一个元素不能回答：
- 它增加了哪种张力
- 它强化了哪种态度
- 它替换了哪种节奏

那它就不该存在。

### 3. 情绪函数优先于物件字面复制
从对标账号转译到猫图腾时，优先保留：
- 物件的情绪功能
而不是：
- 物件的字面身份

例如：
- 对标里是“空桌面”
- 猫图腾可以转译成“被无视的提醒物件”
只要两者都在服务同一个情绪方向。

### 4. 角色永远不是装饰物的附庸
前景可以制造张力，但不能成为主角。中景的猫必须始终是：
- 人格核
- 态度核
- 观众代入的主承载体

### 5. 留白不是空，而是减少噪音
空桌面、空墙面、少物件，不是为了“极简风格”本身，而是为了：
- 不让无关信息分散情绪感知
- 让观众更快完成投射

## Prompt 写法：导演语言与摄影语言

### 导演语言（意图驱动）
导演语言回答的是：
- 观众应该感到什么
- 每个元素为什么存在
- 情绪如何在前/中/后景中流动

这类语言适合：
- 参考源头拆解
- 作战前讨论
- 策划者AI内部推理

### 摄影语言（物理参数驱动）
生成工具最终更容易理解的是：
- 机位与构图
- 物件位置
- 光线方向
- 色温对比
- 角色姿态
- 前后景关系

因此标准工作流应为：

**导演语言（意图） → 翻译为摄影语言（可执行描述） → 投喂工具**

注意：
- 不要把“观众会感到什么”原样直接当作工具能理解的语句
- 要把它翻译成可见的物理关系
- 工具负责生成画面，人类负责判断情绪是否真的被交付出来

## 拆解模板（用于参考源头 / 作战前分析）

### 一句话情绪句
- 观众应该感到：

### 前景
- 物件：
- 情绪功能：
- 为什么它必须存在：

### 中景
- 角色状态：
- 情绪功能：
- 为什么这个体态能成立：

### 后景
- 环境：
- 情绪功能：
- 它在替换哪种节奏：

### 噪音检查
- 画面里有没有任何一个元素不改变情绪输出？
- 如果有，删除。

## 当前边界

以下判断尚未升级为跨赛道通用结论：
- 该三层公式是否适用于荒诞解压型
- 是否适用于行为嘴替型
- 是否适用于反内卷体面型中的“空无压迫感”路线

当前仅可确认：
**它对赛道1（独立自足 / 边界感疗愈）是高适配、可执行、可拆解的视觉叙事公式。**
