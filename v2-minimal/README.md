# 猫图腾 v2-minimal

## 核心摘要
- 这不是文档整理版，而是从账号内核重新长出来的最小操作系统。
- 这套结构只保留4个问题：账号是什么、内容怎么做、怎么判断成立、现在在验证什么。
- 任何新AI只要读完这4个文件，就应该能进入工作状态。
- `tools/` 不是账号本体知识，只是执行辅助层。

## 阅读顺序
1. `01_账号是什么.md`
2. `02_一条内容怎么做.md`
3. `03_什么样的内容算成立.md`
4. `04_现在在验证什么.md`
5. 按需进入 `tools/`

## 为什么只有4个主文件
- 因为猫图腾要成功，最少只需要知道4件事。
- 多出来的角色说明、模板、路由，都不是账号本体知识。
- 它们只在执行时有用，不应该占据主树。

## 当前角色映射
- 用户 = 决策者
- 协调者AI = 跨平台消息传递与流程协调
- L / Claude = 策划者AI
- T / GPT = 评估者AI + GitHub执行者
- GE / Gemini = 执行者AI（当前仅负责 NB2 图片生成）

## 进入工作前的最低要求
- 不要先读归档文档。
- 不要先读角色长文档。
- 先把4个主文件读完，再决定自己需要哪些辅助信息。
